状态码

  • HTTP状态码–>用以表示网页服务器超文本传输协议响应状态的三位数字代码。

常见状态码：

  • 1xx：消息(请求已被接受，需要继续处理。)
  • 2xx：成功访问（200OK）
  • 3xx：重定向（302临时转移至新的url）
  • 4xx：请求错误（404Not Found）
  • 5xx:服务器错误（502Bad Gateway）

with用法

with语句–> 核心：上下文管理器 遵循上下文管理协议，涉及两个方法： enter：进入代码块前执行，返回资源对象（如文件）并赋值给as后的变量。 exit：代码块执行完毕或发生异常时触发，负责关闭资源、释放锁等清理工作。

应用场景：

1.文件操作：替代手动调用close()，即使读取文件发生异常，文件也会被安全关闭：

 with open('text.txt', 'r') as f :
    content = f.read()
        #代码块结束，文件自动关闭

2.线程锁：自动获取和释放锁，避免锁死风险：

import threading
lock = threading.Lock()
with lock:
    # 执行需要安全的操作
    pass
# 锁自动释放

3.数据库连接：许多数据库驱动支持此语法，确保连接或事物被关闭/回滚。

with conn.cursor() as cursor:
     cursor.execute('SELECT * FROM users')
     date = cursor.fetchall()
# 游标自动关闭

4.管理多个上下文（等效于嵌套）

with open('a.txt') as f1, open('b.txt') as f2:
     data1, data2 = f1.read(), f2.read()

响应里面的属性(重点)：

响应内容：

  • text： 以字符串形式输出【获取响应文本】–>针对文本数据
  • content： 以字节流形式输出，若要保存下载图片需使用该属性–>针对媒体数据
  • json()： 获取json数据【后面补充】

关键字参数翻页

开发者工具：负载（分析url的规律）

import requests

# 将 ？ 后面写带的参数 更改为 params 参数传递
url = "https://movie.douban.com/top250"
headers = {
    "user-agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/141.0.0.0 Safari/537.36 Edg/141.0.0.0",
}
# ?start=0&filter=
params = {
    "start": "25",
    "filter": "",
}
# params 通过关键字参数传递查询字符串参数，参数类型是字典
response = requests.get(url, headers=headers, params=params)
print(response.text)

携带cookie

  • cookie：是一个"小型文本文件"，是某些网站为了辨别用户身份，进行Session跟 踪而储存在用户本地终端（浏览器）上的数据（通常经过加密），由用户客户端计算机暂时或永久保存的信息。

携带cookie的好处

  • 能够访问登录后的页面
  • 能够实现部分反反爬
    • 一套cookie对应的是一个个人用户的信息，请求太频繁会被识别出爬虫。——>封号

session模块

  • 用于处理HTTP会话的模块（管理会话数据），它提供了一个类似字典的接口来管理会话数据。通过session对象，可以在多个请求之间共享cookie
  • 使用方法：
# 创建session对象
s = requests.Session()
# 发起请求
s.get()
# 请求方与requests类似。

IP代理（收费，免费的基本用不了）

  • IP 全称“互联网协议地址”， 也称IP地址， 是分配给上网设备的数字标签。

  • 当在爬某个网站的时候， 如果对目标网站发起了大量请求，导致对方将我们的电脑的公网IP封锁了，此时我们再发起请求就无法再获取到数据。

解决方法： 使用代理IP（利用代理IP对网站发起请求）

  • 本地用户——>代理IP（中间人）——>网站读取到的IP地址
  • 本地用户<——代理IP（相当于一个网站）<——（响应）网站返回的数据

使用代理IP：

  • 利用requests方法中的proxies参数：
# 设置代理
proxies = {
    “http”: "http://【ip】:【端口号】"， 
    “https”: "http://【ip】:【端口号】"
    # requests会自动根据不同的网站选择对应的代理（key:http/https）
}
# 添加代理发送请求
requests。get(url, headers=headers, proxies=proxies)

代理IP的获取（如：天启）

代理IP账号不能随意给他人使用

  • 筛选免费的代理IP

    1. 爬取

      • 定位数据请求 分析翻页的时候尽量不要分析特殊页【1， 最后一页】
      • 发起请求

    2. 提取数据 所有对提取数据的分析，都需要分析【响应文本】。 对响应文本不要有任何改变 匹配：

       - 搜索匹配内容的关键字，确定位置<br>

 - 写匹配表达式<br>

     `"ip": "103.17.246.60"` 

     用 (.*?) 替代需要的内容<br>

     `"ip": "(.*?)"`<br>

     `"ip": "(.*?)".*?"port": "(.*?)"`<br>

     匹配到的内容需要留下来的加括号，不需要留下来的不加括号。

    3. 筛选

      • 用获取到的 ip 向网站发送请求
      • 报错【如果报错，失效】 异常处理
      • 超时

      只有出现问题了才重试

示例：

import requests
import re

headers = {
    'Accept': 'text/html,application/xhtml+xml,application/xml;q=0.9,image/avif,image/webp,image/apng,*/*;q=0.8,application/signed-exchange;v=b3;q=0.7',
    'Referer': 'https://www.kuaidaili.cn/free/fps/2',
    'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/128.0.0.0 Safari/537.36',

}
url = 'https://www.kuaidaili.cn/free/fps/1'
response = requests.get(url,headers=headers, timeout=2)

# 用正则表达式匹配response.text中的IP与端口
res = re.findall('"ip": "(.*?)".*?"port": "(.*?)"', response.text)
# print(res)
for ip, port in res:
    print(f"{ip}:{port}") # 规定IP与端口号的输出格式
    url = "https://lloyal-sudo.github.io/lloyal-sudo01/" # 测试IP地址是否有效的网站
    proxies = {
        "http": f"http://{ip}:{port}",
        "https": f"http://{ip}:{port}",
    }
    try:
        resp = requests.get(url, headers=headers, proxies=proxies, timeout=2)
        # 设立超时时间参数
        print(f"代理IP：{ip}:{port}有效")
    except:
        print(f"代理IP：{ip}:{port}无效")

使用超时参数能够加快我们整体的请求速度，但是在正常的网页浏览过程中，如果发生速度很慢的情况，则需要刷新页面(在代码中实现)

在代码中刷新请求：

# 安装retrying ：pip install retrying (或 python -m pip install retrying) 若两个命令都不行的话就使用镜像源地址
from retrying import retry
@retry(stop_max_attempt_number=3)# 设置重试次数（只有出现错误才会重试）
# 类似于 @函数名称 写法 =》装饰器<br>
# 装饰器装饰的是函数或者方法

requests的其他参数

  • 重定向
    • 浏览器的请求从当前URL自动转向另一个目标地址，其核心目的是在不中断用户体验的前提下，实现资源的重新定位或访问 例如：
      • http://www.baidu.com
      • 重定向后会自行修改为： https://www.baidu.com
    • 取消重定向：
      • response = requests.get(url, allow_redirects=False)
