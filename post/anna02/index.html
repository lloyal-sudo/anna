状态码

  • HTTP状态码–>用以表示网页服务器超文本传输协议响应状态的三位数字代码。

常见状态码：

  • 1xx：消息(请求已被接受，需要继续处理。)
  • 2xx：成功访问（200OK）
  • 3xx：重定向（302临时转移至新的url）
  • 4xx：请求错误（404Not Found）
  • 5xx:服务器错误（502Bad Gateway）

with用法

with语句–> 核心：上下文管理器 遵循上下文管理协议，涉及两个方法： enter：进入代码块前执行，返回资源对象（如文件）并赋值给as后的变量。 exit：代码块执行完毕或发生异常时触发，负责关闭资源、释放锁等清理工作。

应用场景：

1.文件操作：替代手动调用close()，即使读取文件发生异常，文件也会被安全关闭：

 with open('text.txt', 'r') as f :
    content = f.read()
        #代码块结束，文件自动关闭

2.线程锁：自动获取和释放锁，避免锁死风险：

import threading
lock = threading.Lock()
with lock:
    # 执行需要安全的操作
    pass
# 锁自动释放

3.数据库连接：许多数据库驱动支持此语法，确保连接或事物被关闭/回滚。

with conn.cursor() as cursor:
     cursor.execute('SELECT * FROM users')
     date = cursor.fetchall()
# 游标自动关闭

4.管理多个上下文（等效于嵌套）

with open('a.txt') as f1, open('b.txt') as f2:
     data1, data2 = f1.read(), f2.read()

响应里面的属性(重点)：

响应内容：

  • text： 以字符串形式输出【获取响应文本】–>针对文本数据
  • content： 以字节流形式输出，若要保存下载图片需使用该属性–>针对媒体数据
  • json()： 获取json数据【后面补充】

请求参数

反爬：爬取数据时，网站的阻碍。

措施：

  1. 添加请求头:检测请求信息
  • ua
  • referer
  • cookie
  • 其他的补全

如果所给的请求头有错误， 那么请求一定出错，如果是某一个请求头没有，不一定出错。

import requests
url = "https://www.baidu.com/"
headers = {
     "键"："值" # 开发者工具中寻找，也可自己收集添加到字典，后面直接拿来用。
}
response = requests.get(url, headers=headers)
print(response.text)

cookie可在请求头中携带，也可以单独携带。 如：

cookies = {
     "key": "value"
}
# 调用
response = requests.git(url, headers=headers, cookies=cookies)

在一个网站中，不同页面的url有着一定的规律—>使用循环与f函数实现url的转换

  • 翻页：
  需要分析不同页面之间，数据的规律。

关键字参数翻页

开发者工具：负载（分析url的规律）

import requests

# 将 ？ 后面写带的参数 更改为 params 参数传递
url = "https://movie.douban.com/top250"
headers = {
    "user-agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/141.0.0.0 Safari/537.36 Edg/141.0.0.0",
}
# ?start=0&filter=
params = {
    "start": "25",
    "filter": "",
}
# params 通过关键字参数传递查询字符串参数，参数类型是字典
response = requests.get(url, headers=headers, params=params)
print(response.text)

补充：url编码/解码 

from urllib import parse
# url编码：
res = parse.quote("壁纸")
print(res)
# url解码：
print(parse.unquote("%E5%A3%81%E7%BA%B8"))

表单参数

  1. 查询字符串参数：可直接在url通过？携带。可以出现在get&post请求方式中，可以在开发者工具：负载中看到相关信息， 比如：username&passward
  2. 表单参数：相关数据在url后面看不到。
  • 只会出现在post请求中。
  • 参数形式：字典
  • 携带方式（只有一种）：
# data参数携带
requests.post(url, headers=headers, data={"key":"value"})

使用正则表达式进行批量修改

  1. 将复制过来的内容改成 key value 在同一行，中间要用东西隔开
        accept: */*
        accept-encoding: gzip, deflate, br, zstd
  1. 按 ctrl + r【替换】
  2. 复制任意一行到上面输入框
  3. 修改匹配规则【正则表达式】
   .*?: .*
   (.*?): (.*)
       # 一个小括号代表一个组
       # 如果要用第几个组里面的内容，就用$序号
   $1 $2
   '$1': '$2',
   'key': 'value',
   shift + 4 => 英文符号
  1. 点击最右边 的 “.*”
